Introduction

ConvoForm is a modern, conversational form component for React applications that transforms traditional forms into engaging chat-like conversations. It provides a more interactive and user-friendly way to collect information from your users.

Key features include:

  • Chat-like interface for form inputs
  • Support for various input types (text, email, number, select, etc.)
  • Customizable appearance and behavior
  • Data export capabilities
  • EmailJS integration for sending form data
  • Easy embedding in any HTML page

Installation

Using the Pre-built Version

The easiest way to use ConvoForm is to include the pre-built files in your HTML:

<link rel="stylesheet" href="path/to/convoform/dist/assets/index-[hash].css">
<script src="path/to/convoform/dist/assets/index-[hash].js" defer></script>

Using the Source Code

If you want to customize or integrate ConvoForm into your React project:

  1. Copy the source code into your project
  2. Install dependencies: 
    npm install
  3. Start the development server: 
    npm run dev
  4. Build for production: 
    npm run build

Usage

Basic Usage

To use ConvoForm in your React application:

import FormEngine from './components/FormEngine';

function App() {
  return (
    <div className="App">
      <FormEngine 
        configUrl="path/to/config.json"
        onComplete={(data) => console.log(data)}
      />
    </div>
  );
}

Configuration

ConvoForm is configured using a JSON file. Here's an example configuration:

{
  "botName": "FormBot",
  "botAvatar": "https://example.com/bot-avatar.png",
  "userAvatar": "https://example.com/user-avatar.png",
  "welcomeMessage": "Hello! I'm here to help you fill out this form.",
  "questions": [
    {
      "id": "name",
      "question": "What is your name?",
      "type": "text",
      "required": true
    },
    {
      "id": "email",
      "question": "What is your email address?",
      "type": "email",
      "required": true
    },
    {
      "id": "age",
      "question": "How old are you?",
      "type": "number",
      "required": true
    }
  ],
  "thankYouMessage": "Thank you for completing the form!",
  "emailJS": {
    "serviceId": "your_service_id",
    "templateId": "your_template_id",
    "publicKey": "your_public_key"
  }
}

Input Types

ConvoForm supports the following input types:

Type Description Example
text Single-line text input "type": "text"
email Email address input with validation "type": "email"
number Numeric input "type": "number"
select Dropdown selection "type": "select", "options": ["Option 1", "Option 2"]
date Date picker "type": "date"
tel Telephone number input "type": "tel"

Embedding in HTML

Step-by-Step Guide

Follow these steps to embed ConvoForm in any HTML page:

  1. Include the ConvoForm CSS and JavaScript files: 
    <link rel="stylesheet" href="path/to/convoform/dist/assets/index-[hash].css">
<script src="path/to/convoform/dist/assets/index-[hash].js" defer></script>
  2. Create a container element where you want the form to appear: 
    <div id="convoform-container"></div>
  3. Initialize the form with your configuration: 
    <script>
  document.addEventListener('DOMContentLoaded', function() {
    // Wait for ConvoForm to be loaded
    setTimeout(function() {
      if (window.ConvoForm) {
        // Initialize with configuration
        window.ConvoForm.init({
          targetElement: '#convoform-container',
          configUrl: 'path/to/config.json',
          onComplete: function(data) {
            console.log('Form completed with data:', data);
            // You can send this data to your server or process it as needed
          }
        });
      }
    }, 500);
  });
</script>

Note: You can find a complete working example in the Demo/demo-embed.html file included in this package.

Embedding Options

The ConvoForm.init() function accepts the following options:

Option Type Description Required
targetElement String CSS selector for the container element Yes
configUrl String URL to the configuration JSON file Yes
onComplete Function Callback function that receives the form data No
theme Object Custom theme options No

Customization

ConvoForm can be customized in several ways:

Appearance

You can customize the appearance by modifying the CSS or by providing a custom theme object when initializing ConvoForm:

window.ConvoForm.init({
  targetElement: '#convoform-container',
  configUrl: 'path/to/config.json',
  theme: {
    primaryColor: '#4a6cf7',
    backgroundColor: '#f5f5f5',
    fontFamily: 'Arial, sans-serif'
  }
});

Behavior

You can customize the behavior by modifying the configuration JSON file. For example, you can change the welcome message, add more questions, or modify the thank you message.

Data Export

ConvoForm provides several ways to export the collected data:

JavaScript Callback

You can access the form data through the onComplete callback:

window.ConvoForm.init({
  targetElement: '#convoform-container',
  configUrl: 'path/to/config.json',
  onComplete: function(data) {
    console.log('Form completed with data:', data);
    // Send data to your server or process it as needed
  }
});

EmailJS Integration

ConvoForm can automatically send the form data to your email using EmailJS. See the EmailJS Integration section for more details.

EmailJS Integration

ConvoForm integrates with EmailJS to send form data to your email. To use this feature:

Important: By default, form results are logged in the console. To enable email notifications, integrate your own EmailJS account as described below.

  1. Sign up for an account at EmailJS
  2. Create a service and template
  3. Add your EmailJS configuration to your ConvoForm configuration file: 
    "emailJS": {
  "serviceId": "your_service_id",
  "templateId": "your_template_id",
  "publicKey": "your_public_key"
}

Warning: Your EmailJS public key will be visible in the configuration file. Make sure to set up proper security measures on the EmailJS side to prevent abuse.

Troubleshooting

Common Issues

ConvoForm is not appearing on the page

Check the following:

  • Make sure the CSS and JavaScript files are correctly included
  • Check that the container element exists in the DOM
  • Check the browser console for any errors
  • Make sure the configuration file is accessible

Form data is not being sent

Check the following:

  • Make sure the EmailJS configuration is correct
  • Check the browser console for any errors
  • Verify that the form is being completed correctly
